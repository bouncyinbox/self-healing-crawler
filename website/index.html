How It Works
Two-path extraction pipeline

Optimized for cost and speed. The LLM is only called when drift is detected.

1

Change Detection

Dual-signal drift detection using Jaccard similarity on DOM 3-gram shingles and perceptual hashing (pHash) of page screenshots. Thresholds: <0.85 structural or <0.80 visual triggers healing.

2

Cache Lookup

If no drift detected, cached CSS selectors are applied via BeautifulSoup/lxml for near-zero latency extraction (~3ms). Thread-safe JSON cache with 7-day TTL and canary validation.

3

LLM Extraction

On cache miss or drift: content-aware HTML truncation finds <main> or product markup, sends to Claude for semantic extraction. Generates new CSS selectors for caching.

4

Validate & Store

Pydantic schema validates all fields with lenient parsers (extracts "$29.99" from messy HTML). Results stored in async SQLite with full audit trail.

Features
Built for production

Every component designed for reliability, cost efficiency, and observability.

⚡

Dual-Signal Drift Detection

DOM shingle fingerprinting + perceptual image hashing catches both structural and visual changes.

🤖

LLM-Powered Extraction

Claude extracts structured data semantically and generates CSS selectors for future fast-path reuse.

🚀

Write-Behind Cache

Thread-safe selector cache with TTL, canary validation, and write-behind buffering to minimize I/O.

📊

Canary Validation

Auto-invalidates selectors when null rate exceeds 25% over 5+ samples, catching gradual drift.

🛠

Content-Aware Truncation

Finds <main>, product schema markup, or content regions before sending to LLM. Fewer tokens, better results.

📑

Full Audit Trail

Every extraction logged with method, drift severity, tokens used, and selectors applied. Async SQLite with WAL mode.

Quick Start
Up and running in 60 seconds

No API key needed for the built-in demo with mock HTML files.

1. Install

Clone the repository and install dependencies. Python 3.11+ required.

# Clone and install git clone https://github.com/user/self-healing-crawler cd self-healing-crawler pip install -e ".[dev]" # Run the demo (no API key needed) python -m crawler demo

2. Crawl live URLs

Set your Anthropic API key and crawl any product page.

# Set API key export ANTHROPIC_API_KEY="sk-ant-..." # Crawl a live product page python -m crawler crawl "https://example.com/product" # Or crawl a local HTML file python -m crawler local product_v1.html # View cache and extraction stats python -m crawler stats

3. Define your schema

Pydantic models with lenient field parsers handle messy real-world HTML.

from crawler.schema import ProductSchema # Built-in schema handles: # - Price: "$29.99" -> 29.99 # - Rating: "4.5 out of 5" -> 4.5 # - Stock: "In Stock" -> True # - Reviews: "(2,847 reviews)" -> 2847 schema = ProductSchema( title="Pro X500 Headphones", price="$299.99", currency="USD", in_stock="In Stock", rating="4.7 out of 5", review_count="(2,847 reviews)", sku="TG-X500-BLK", brand="TechGear", ) print(schema.price) # 299.99 print(schema.in_stock) # True print(schema.rating) # 4.7
